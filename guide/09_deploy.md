## Docker构建镜像部署
```
cd sbin
sh build.sh

docker push xxxx

docker pull xxxx

docker run -d --name=guru -p 3500:3500 xxxx
```

## 常见web前后端nginx配置

```
  location / {
    root /var/www/;
  }

  location /api {
    proxy_pass http://localhost:3500/;
  }

```
后端接口通常通过端口转发到对应服务，前端本地开发也可以`/api`使用服务器代理
